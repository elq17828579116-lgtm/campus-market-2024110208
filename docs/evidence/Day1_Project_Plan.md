# 项目规划 - 校园轻集市

## 页面清单

| 页面 | 路由 | 功能描述 |
|------|------|----------|
| 首页 | `/` | 商品推荐、分类导航、搜索入口 |
| 商品列表 | `/products` | 按分类筛选商品，支持搜索 |
| 商品详情 | `/product/:id` | 商品信息展示、加入购物车 |
| 登录 | `/login` | 用户登录 |
| 注册 | `/register` | 用户注册 |
| 购物车 | `/cart` | 购物车管理 |
| 订单确认 | `/checkout` | 下单确认页 |
| 个人中心 | `/profile` | 用户信息管理 |
| 我的订单 | `/orders` | 订单列表与状态 |
| 发布商品 | `/publish` | 发布二手商品 |

## 功能模块

| 模块 | 子功能 | 优先级 |
|------|--------|--------|
| 用户模块 | 登录、注册、个人信息管理 | P0 |
| 商品模块 | 商品浏览、搜索、分类筛选、详情 | P0 |
| 购物车模块 | 添加/删除商品、数量修改 | P1 |
| 订单模块 | 创建订单、订单列表、状态管理 | P1 |
| 发布模块 | 商品发布、编辑、下架 | P1 |
| 个人中心 | 基本信息、地址管理 | P2 |

## 开发顺序

1. **Day2**: 路由设计 + 页面骨架搭建（登录、首页、商品列表）
2. **Day3**: 用户认证模块 + API 对接
3. **Day4**: 商品详情 + 购物车功能
4. **Day5**: 订单流程（下单、支付模拟）
5. **Day6**: 个人中心 + 商品发布
6. **Day7**: 联调测试 + 优化完善

## 开发重点

- 路由守卫实现登录验证
- Pinia 状态管理购物车和用户状态
- API 请求封装与错误处理
- 组件复用设计（商品卡片、列表等）
- 响应式布局适配
